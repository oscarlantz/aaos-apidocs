auditTrailEntry Data Type

An AuditTrailEntry represents one audit trail entry collected from a CLIQ element (key or cylinder). The entry has a unique identity that can be used for separating two entries from each other. The entry contains information about the element's interaction with another CLIQ element.

E.g. If an audit trail entry is read from a CLIQ key, it may contain entries created from interaction with Cylinders (access), C-keys (programming), or Remote Programming Devices (programming). If an audit trail is collected from a cylinder, it may contain entries created from interaction with Keys (access), C-keys (programming), or Local Programming Devices (programming). Other element interactions may also occur.

Audit trails come in two flavours:

  • NormalAuditTrailEntry - For interaction with elements in same MKS
  • ForeignAuditTrailEntry - For interaction with elements in foreign MKS
Normal entries contain information about the other element, while foreign audit trail entries only contain limited information needed to identify the element in the foreign system.
