getKeyAuditTrail Operation

Query for audit trail entries. The dates in a query are creation dates that are assumed to be in MKS time zone.

getKeyAuditTrail method returns correct creation dates provided that CWM server time zone (Windows time zone) was the same at the audit trail import time and the time when the web service is used.

If CWM server time zones are different then there will be several hours discrepancies. We have decided that the discrepancies will remain due to the fact that fixing it will require running a sql script on all production databases. Otherwise the web service won't be consistent for all users.

The following table shows an example of discrepancies.

CWM import 1st time zone - CWM server time zone during the first import of audit trail and save to the database

CWM read 2nd time zone - after restarting CWM server with a different time zone

The event happened at 23:02 Asia/Shanghai (MKS local time)

MKS time zone MKS local time CWM import 1st time zone DB record time = MKS + (CWM import 1st time zone - UTC) CWM import 1st time CWM read 2nd time zone CWM read 2nd time = DB record time - (CWM read 2nd time zone - UTC)
Asia/Shanghai 23:02 28 Feb Europe/Kaliningrad UTC+02 01:02 1st March 23:02 28 Feb Asia/Baku UTC+04 21:02 28 Feb
Asia/Shanghai 23:02 28 Feb Europe/Kaliningrad UTC+02 01:02 1st March 23:02 28 Feb America/Los Angeles UTC-08 09:02 1st March

getAuditTrails Operation

Query for all audit trails entries (key and cylinder) that were imported within the given date interval. The date interval is exclusive for the 'from' date and inclusive for the 'to' date. If the 'to' date is omitted, the current date and time in the system-specific timezone will be used.

An example of usage would be to query for all events between 2001-01-01T00:01 and 2001-06-01T00:01. This will return all events that were imported after the 'from' date and before or on the 'to' date. If the pagination maximum is set to 1000 and more than 1000 events are found, only the first 1000 events will be returned.

The import date is defined as the date when the audit trails were collected (imported) into the system in CWM server time zone, as opposed to the creation date which specifies when they were originally created in MKS time zone.

Input Parameters
name type description
importDateInterval dateIntervalWithTimeZone Audit trail import date interval. The date interval is exclusive for the 'from' date and inclusive for the 'to' date. If the 'to' date is omitted, the current date and time in the system-specific timezone will be used.
includePartiallyProcessed boolean  
excludeEntriesFromKeysWithoutTimeSupport boolean If specified as true, then only entries where key supports time are returned.
pagination pagination used to specify the starting row and maximum amount of rows to return, allowing for iteration through large result sets. The maximum allowed value is 10000.
Return Value
type description
auditTrailDetails Audit trail items matching the given import date interval, sorted by oldest to newest import date.
Faults
name type conditions
SystemException SystemException Thrown if an unrecoverable error occurs.
InvalidDataException InvalidDataException Thrown if date interval or pagination is missing or invalid or if pagination maximum results exceeds 10000.

getAccessProfilesForKey Operation

A query for all access profiles associated with a specific key. A key can be associated with an access profile directly (hasPermanentAssignment = true). 'Directly' means that there is an explicit association between an access profile and a key for an indefinite period of time. 'Indefinite' means until an access profile is unassociated from a key by a user. A key can be associated with an access profile indirectly via a temporary access group (hasPermanentAssignment = false). A temporary access group is a set of multiple keys and multiple access profiles associated together for a period of time (accessTimeInterval in the response) A key or an access profile can be included in multiple temporary access groups. If a key is associated to an access profile both directly and indirectly then it manifests as (hasPermanentAssignment = true).

Input Parameters
name type description
keyIdentity string unique identity (uuid) of a key.
Return Value
type description
assignedAccessProfile all access profiles associated to a key
Faults
name type conditions
SystemException SystemException Thrown if an unrecoverable error occurs.
IdentityNotFoundException IdentityNotFoundException Thrown if no key exists with the given identity.
InvalidDataException InvalidDataException
